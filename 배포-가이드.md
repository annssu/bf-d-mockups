# 🚀 웨잇리스트 배포 가이드 (Supabase 저장 + Vercel 공개)

신청 폼을 인터넷에 올리고, 들어온 명단을 베프디 Supabase에 쌓는 방법이에요.
코딩은 제가(클로드) 다 해놨고, **수현님은 복사-붙여넣기 몇 번**만 하면 돼요.

> 구조: 신청자가 폼 제출 → Vercel의 작은 서버(`/api/waitlist`)가 받음 → Supabase에 저장
> 키(비밀번호 같은 값)는 **Vercel 설정에만** 넣고 코드엔 안 남겨요. 안전해요.

---

## 1단계 · Supabase에 명단 표(table) 만들기

1. https://supabase.com 접속 → 베프디 프로젝트 열기 (없으면 "New project")
2. 왼쪽 메뉴에서 **SQL Editor** 클릭 → **New query**
3. 아래를 **그대로 복사해서 붙여넣고** 오른쪽 아래 **Run** ▶️ 누르기:

```sql
create table if not exists waitlist (
  id          bigint generated always as identity primary key,
  created_at  timestamptz default now(),
  name        text,
  phone       text,
  reason      text,
  linkedin    text,
  source      text
);
-- 보안: 아무나 "신청(추가)"만 가능, 명단 열람은 운영진(키 가진 사람)만
alter table waitlist enable row level security;
```

> "Success. No rows returned" 비슷한 메시지가 뜨면 성공이에요.
> 서버 함수는 service_role 키를 쓰기 때문에 별도 insert 정책 없이도 저장돼요.

---

## 2단계 · Supabase 키 2개 복사해두기

왼쪽 메뉴 맨 아래 **⚙️ Project Settings → API** 로 가서 2개를 메모장에 복사:

| 이름 | 어디에 있나 | 쓸 곳 |
|------|------------|-------|
| **Project URL** | "Project URL" 칸 (`https://....supabase.co`) | 아래 `SUPABASE_URL` |
| **service_role** | "Project API keys" → `service_role` (👁️ 눌러서 보기) | 아래 `SUPABASE_SERVICE_KEY` |

> ⚠️ `service_role` 키는 **비밀번호급**이에요. 채팅·코드·깃허브에 절대 붙여넣지 말고,
>   3단계의 Vercel 설정 칸에만 넣으세요. 혹시 어디 노출되면 Supabase에서 재발급하세요.

---

## 3단계 · Vercel에 올리기

1. https://vercel.com 접속 → 깃허브 계정으로 로그인
2. **Add New… → Project** → 깃허브 저장소 목록에서 **`bf-d-mockups`** 선택 → Import
3. 설정은 건드릴 것 없어요 (자동 감지). 단, **Environment Variables** 섹션을 펼쳐서 아래 3개 추가:

| Name | Value |
|------|-------|
| `SUPABASE_URL` | 2단계에서 복사한 Project URL |
| `SUPABASE_SERVICE_KEY` | 2단계에서 복사한 service_role 키 |
| `WAITLIST_START_NUMBER` | `47` (대기순번 시작 숫자, 원하면 바꿔도 됨) |

4. **Deploy** 클릭 → 1~2분 기다리면 주소가 나와요:
   `https://bf-d-mockups.vercel.app/waitlist-a.html`

---

## 4단계 · 잘 되는지 확인

1. 나온 주소 + `/waitlist-a.html` 열기
2. 폼에 테스트로 이름·전화 넣고 **등록하기** → "등록 완료!" 화면이 뜨면 성공 🎉
3. Supabase로 돌아가 왼쪽 **Table Editor → waitlist** 열면, 방금 넣은 줄이 보여요.

> 💡 신청 명단 내보내기: Table Editor 우측 상단에서 CSV로 다운로드 가능해요.

---

## 자주 묻는 것

- **"등록 중 문제가 생겼어요"가 떠요** → Vercel 환경변수 3개 철자/값이 맞는지 확인하고, 다시 Deploy(재배포) 하세요.
- **로컬(내 컴퓨터)에서 테스트하면 저장이 안 돼요** → 정상이에요. 내 컴퓨터에선 디자인 미리보기(demo)만 되고, 진짜 저장은 Vercel에 올라간 주소에서만 돼요.
- **A안/B안 둘 다 같은 명단에 쌓여요?** → 네. 둘 다 같은 `/api/waitlist`로 보내서 한 표에 모여요. (`source` 칸으로 어느 페이지인지 구분돼요.)
